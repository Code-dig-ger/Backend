Title

To get the user's standing in the list w.r.f their friends or in user in that list.

URL

GET
/lists/{slug}/stats/standing

Headers

Header Name Required Type Description
Authorization true string Bearer {Token}

URL Parameters

For GET Request. If URL params exist, specify them in accordance with name mentioned in URL section. Mention if is required or optional. Document data constraints.

Param Name Required Type Description
friend false boolean friend=True means rank with respect to their friends

Success Response

Returns ranking of the users in that list

Status Code: 200

Sample Response: 

            
                {
                    "status":"OK",
                    "result":[
                       {
                          "user":1,
                          "rank":1,
                          "problems_solved":2
                       },
                       {
                          "user":2,
                          "rank":2,
                          "problems_solved":1
                       }
                    ]
                 }

Error Response

Reason of Error: If the user is not Authentication.

Status Code: 400 BAD REQUEST

Error Response: 

            
                {
                    "status": "FAILED",
                    "error": "Authentication credentials were not provided"
                }

Just a sample call to your endpoint in a runnable format ($.ajax call or a curl request) - this makes life easier and more predictable. 

            
              curl -X GET "https://api.codedigger.tech/lists/testinglist_topicwise/stats/standing"
